Diferença entre BEM (Block Element Modifier) e SMACSS (Scalable and Modular Architecture for CSS)

BEM: enfatiza uma nomenclatura específica para classes CSS
SMACSS: organiza estilos em categorias como Base, Layout e Módulos.


Pilares do SMACSS:

  1. Base: Este é o nível mais baixo da hierarquia, onde definimos estilos para elementos HTML padrão, como tags de parágrafo, cabeçalhos, etc. São estilos gerais que se aplicam globalmente ao projeto.
  2. Layout: Aqui são definidos os estilos para os principais componentes de layout da página, como barras laterais, áreas de conteúdo principal, cabeçalho, rodapé, etc. Estes estilos estruturam a aparência geral da página.
  3. Módulos: Os módulos são componentes independentes e reutilizáveis da interface do usuário, como menus de navegação, cartões de produto, caixas de diálogo, etc. Eles são projetados para serem autossuficientes e facilmente transferíveis entre diferentes partes do projeto.
  4. Estado: Este pilar trata de como os estilos podem mudar com base em diferentes estados do elemento, como hover, foco, ativo, etc. Ele ajuda a definir como os componentes interagem com o usuário em diferentes contextos.
  5. Temas (Themes): Embora não seja sempre aplicável, o SMACSS sugere separar os estilos que são específicos para diferentes temas do projeto, como esquemas de cores diferentes ou estilos de fonte.

Outros Conceitos Importantes:

  • Cascata e Especificidade: O SMACSS promove o uso eficiente da cascata CSS e evita a especificidade excessiva, priorizando classes e evitando seletores muito específicos.
  • Reutilização: A reutilização de estilos é enfatizada para evitar duplicação de código e facilitar a manutenção.
  • Organização Modular: Os estilos são organizados em arquivos separados e agrupados por função (base, layout, módulos, etc.) para facilitar a localização e a manutenção.
  • Nomenclatura Significativa: O SMACSS sugere uma nomenclatura significativa para classes CSS, tornando mais fácil entender o propósito de cada estilo e promovendo uma melhor compreensão do código.

Este é um exemplo simples de como aplicar os conceitos do SMACSS em uma aplicação web.

Botão Primário Botão Secundário

Explicação:

  1. Base: Definimos estilos básicos para o corpo (body) da página, como fonte, cores de fundo e texto, e removemos as margens e preenchimentos padrão.
  2. Layout: Criamos uma classe .container para centralizar o conteúdo da página e limitar sua largura máxima.
  3. Módulos: Criamos uma classe .button para estilizar botões comuns, definindo propriedades como padding, cor de fundo e texto, e borda arredondada.
  4. Estado: Utilizamos pseudo-classes :hover e :focus para alterar a cor de fundo do botão quando o cursor do mouse está sobre ele ou quando recebe foco.
  5. Temas (Themes): Definimos uma variação do botão com a classe .button.secondary, que possui uma cor de fundo diferente para indicar um estilo secundário.
  6. Nomenclatura Significativa: Utilizamos a classe .header para estilizar o cabeçalho da página, fornecendo um tamanho de fonte maior e uma margem inferior para separá-lo do restante do conteúdo.
