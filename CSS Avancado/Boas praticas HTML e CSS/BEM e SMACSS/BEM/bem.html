Diferença entre BEM (Block Element Modifier) e SMACSS (Scalable and Modular Architecture for CSS)

BEM: enfatiza uma nomenclatura específica para classes CSS
SMACSS: organiza estilos em categorias como Base, Layout e Módulos.


Método BEM (CSS)

  • FÁCIL: É uma convenção de nomes
  • MODULAR: Código independente e reusável
  • FLEXÍVEL: Use da maneira que preferir
  • Semântica Clara: BEM enfatiza uma semântica clara ao nomear classes CSS.
  • Hierarquia Definida: Estabelece uma hierarquia clara entre blocos, elementos e modificadores.
  • Reutilização de Código: Promove a reutilização de código, reduzindo a duplicação.
  • Escopo Isolado: Minimiza o impacto de estilos em outros elementos da página.
  • Manutenção Simplificada: Facilita a localização e a modificação de estilos específicos.
  • Compatibilidade com Ferramentas: É compatível com várias ferramentas de desenvolvimento.
  • Documentação Facilitada: Torna a documentação do código mais fácil e compreensível.

Resumo dos Pilares:

  • Blocos (Blocks): Entidades de nível mais alto na metodologia BEM.
  • Elementos (Elements): Partes menores e dependentes dos blocos.
  • Modificadores (Modifiers): Usados para alterar o estado ou a aparência de um bloco ou elemento.

Blocos (Blocks)

  • Os blocos são as entidades de nível mais alto na metodologia BEM.
  • Eles representam componentes ou partes independentes e significativas da interface do usuário.
  • Exemplos de blocos podem incluir botões, menus, footer, checkbox, barras laterais, formulários, entre outros.
  • Os blocos são definidos por uma classe CSS única e descritiva que reflete seu propósito e funcionalidade. Ex.: .menu

Elementos (Elements)

  • Os elementos são partes menores e dependentes dos blocos (elemento-filho - Ex.: item de menu, item de lista label do checkbox...).
  • Eles são elementos individuais que fazem parte de um bloco e têm significado apenas dentro desse contexto.
  • Os elementos são representados por classes CSS aninhadas sob a classe do bloco pai.
  • Essas classes geralmente têm o formato (seguindo o exemplo anterior) .menu__item (se usa 2 underline), onde .menu é o nome do bloco pai e item é o nome do elemento específico.

Modificadores (Modifiers)

  • Os modificadores são usados para alterar o estado ou a aparência de um bloco ou elemento.
  • Eles permitem criar variações de um bloco ou elemento sem precisar reescrever todo o CSS.
  • Ex.: disabled, checked, full-width, dark...
  • Os modificadores são representados por classes CSS adicionais aplicadas ao bloco ou elemento base.
  • Essas classes geralmente têm o formato (seguindo o exemplo anterior) .menu--dark (menu com variação dark) ou .menu__item--dark (item do menu com variação dark), onde --dark é o nome do modificador.
